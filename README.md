## Описание

KwasOS — это небольшая операционная система, разрабатываемая с нуля как учебный проект. Система написана на C и ассемблере, поддерживает архитектуру x86_64 и включает в себя:

- **Собственный загрузчик** — загружает ядро с диска в память
- **Ядро** — базовая реализация с обработкой прерываний и командной оболочкой
- **Драйверы устройств** — клавиатура, таймер, VGA, ATA/FAT32
- **Файловая система FAT32** — чтение, запись, форматирование диска
- **Интерактивная командная строка** — набор встроенных команд

> **Внимание!** Система находится в стадии активной разработки и пока не тестировалась на реальном железе. Запускайте только в эмуляторе QEMU!

## Возможности

### Ядро
- 64-битный режим (Long Mode)
- Обработка прерываний (IDT, ISR)
- Программируемый контроллер прерываний (PIC)
- Паника ядра (аналог BSOD)
- Перезагрузка и выключение системы

### Драйверы
- **VGA** — текстовый видеорежим (80×25, 16 цветов)
- **Клавиатура** — ввод с PS/2 клавиатуры
- **Таймер** — подсчёт тиков и uptime
- **ATA** — доступ к жесткому диску
- **FAT32** — файловая система с поддержкой чтения/записи

### Пользовательский интерфейс
- Интерактивная командная оболочка (shell)
- Поддержка цветов в выводе
- История команд (в буфере ввода)

## 🛠 Системные требования

| Компонент | Назначение | Минимальная версия |
|-----------|------------|-------------------|
| **Git** | Клонирование репозитория | Любая современная |
| **NASM** | Сборка загрузчика | 2.14+ |
| **GCC** | Компиляция ядра | С поддержкой `-m64`, `-ffreestanding` |
| **QEMU** | Эмуляция x86_64 | 4.0+ |
| **Xorriso** | Создание ISO-образов | 1.5.0+ |
| **Make** | Автоматизация сборки | 4.0+ |

> **Примечание:** В makefile используется кросс-компилятор `x86_64-elf-gcc` для macOS и Windows. На Linux можно использовать системный `gcc` с флагами `-m64` и `-ffreestanding`.

## Установка зависимостей

### Linux (Ubuntu/Debian)

```bash
# Обновление пакетов
sudo apt update

# Установка базовых инструментов
sudo apt install -y git nasm qemu-system-x86 xorriso build-essential make

# Установка кросс-компилятора (опционально)
sudo apt install -y gcc-x86-64-linux-gnu binutils-x86-64-linux-gnu

# Для 32-битных библиотек (если требуется)
sudo apt install -y gcc-multilib g++-multilib
```

### Linux (Fedora/RHEL)

```bash
# Установка базовых инструментов
sudo dnf install -y git nasm qemu-system-x86 xorriso gcc make binutils

# Кросс-компилятор (опционально)
sudo dnf install -y gcc-x86_64-linux-gnu
```

### Linux (Arch Linux)

```bash
# Установка базовых инструментов
sudo pacman -S git nasm qemu-desktop xorriso gcc make

# Кросс-компилятор (если доступен)
sudo pacman -S x86_64-elf-gcc
```

### macOS

```bash
# Установка Homebrew (если не установлен)
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Установка инструментов
brew install git nasm qemu xorriso make

# Установка кросс-компилятора
brew install x86_64-elf-gcc

# Проверка установки
which x86_64-elf-gcc
```

> Если `x86_64-elf-gcc` недоступен, можно использовать сторонний tap:
> ```bash
> brew tap nativeos/i386-elf-toolchain
> brew install i386-elf-gcc
> ```

### Windows (WSL2)

**Шаг 1: Установка WSL2**

Откройте PowerShell от имени администратора:

```powershell
wsl --install
```

Перезагрузите компьютер и запустите Ubuntu из меню «Пуск».

**Шаг 2: Установка зависимостей внутри WSL**

Выполните инструкции для [Ubuntu/Debian](#linux-ubuntudebian).

**Шаг 3: Запуск QEMU**

Для графического режима установите X-сервер (например, [VcXsrv](https://sourceforge.net/projects/vcxsrv/)) и настройте переменную `DISPLAY`:

```bash
export DISPLAY=:0
```

Или используйте режим без графики:

```bash
make run QEMU_FLAGS="-nographic"
```

### Альтернатива: Docker

Если вы не хотите устанавливать зависимости локально, используйте Docker:

```dockerfile
# Dockerfile
FROM ubuntu:22.04

RUN apt update && \
    apt install -y git nasm qemu-system-x86 xorriso build-essential \
    gcc-x86-64-linux-gnu binutils-x86-64-linux-gnu make

WORKDIR /KwasOS
COPY . .

CMD ["make", "run"]
```

Сборка и запуск:

```bash
docker build -t kwasos .
docker run --rm -it kwasos
```

```bash
git clone https://github.com/frendalkir-alt/KwasOS.git
```


## Архитектура

### Загрузчик (boot.asm)

Загрузчик KwasOS выполняет следующие этапы:

1. **16-битный режим (Real Mode)**
   - Инициализация сегментных регистров
   - Включение линии A20
   - Чтение ядра с диска (128 секторов, 64 КБ)
   - Переход в защищённый режим

2. **32-битный режим (Protected Mode)**
   - Настройка GDT
   - Включение PAE (Physical Address Extension)
   - Настройка таблиц страниц
   - Включение long mode через MSR

3. **64-битный режим (Long Mode)**
   - Загрузка 64-битного GDT
   - Копирование ядра по адресу `0x100000`
   - Передача управления ядру

### Ядро (kernel.c)

Точка входа — функция `kmain()`:

```c
void kmain(void) {
    // 1. Инициализация видеодрайвера
    clear_screen();
    
    // 2. Настройка прерываний (IDT, PIC)
    idt_init();
    pic_remap();
    
    // 3. Инициализация клавиатуры
    init_keyboard();
    
    // 4. Настройка таймера
    timer_init();
    
    // 5. Инициализация ATA и FAT32
    ata_init();
    fat32_init();
    
    // 6. Включение прерываний
    __asm__ volatile ("sti");
    
    // 7. Запуск командной оболочки
    while (1) {
        readline(buffer, sizeof(buffer));
        process_command(buffer);
    }
}
```

### Обработка прерываний

- **IDT (Interrupt Descriptor Table)** — таблица дескрипторов прерываний
- **ISR (Interrupt Service Routines)** — обработчики исключений процессора
- **IRQ (Interrupt Requests)** — обработчики аппаратных прерываний
- **PIC (Programmable Interrupt Controller)** — переназначение IRQ на векторы 0x20–0x2F

### Драйверы

| Драйвер | Файл | Описание |
|---------|------|----------|
| **VGA** | `video.c` | Текстовый режим 80×25, 16 цветов, прямой доступ к памяти 0xB8000 |
| **Клавиатура** | `keyboard.c` | PS/2 скан-коды, буферизация ввода |
| **Таймер** | `timer.c` | PIT (Programmable Interval Timer), подсчёт тиков |
| **ATA** | `ata.c` | Доступ к IDE/SATA дискам через порты |
| **FAT32** | `fat32.c` | Чтение/запись файлов, форматирование |
| **PIC** | `pic.c` | Управление масками прерываний |

## Вклад в проект

Если вы хотите внести свой вклад в разработку KwasOS:

1. **Форкните репозиторий**
2. **Создайте ветку** для вашей функции:
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. **Внесите изменения** и убедитесь, что код компилируется
4. **Протестируйте** в QEMU
5. **Закоммитьте** изменения с понятным сообщением:
   ```bash
   git commit -m "feat: добавить поддержку XYZ"
   ```
6. **Отправьте пул-реквест** в основной репозиторий

### Требования к коду

- Используйте `clang-format` или следуйте стилю проекта
- Добавляйте комментарии для сложных участков кода
- Избегайте утечек памяти
- Тестируйте изменения в эмуляторе перед отправкой
