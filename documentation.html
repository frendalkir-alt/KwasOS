Введение

KwasOS — это учебная операционная система, разрабатываемая энтузиастами с любовью к системному программированию и, конечно, к квасу. Проект создан для погружения в мир низкоуровневой разработки: от загрузчика до простого ядра с базовыми драйверами. KwasOS работает на архитектуре x86_64, использует мультизагрузочный стандарт Multiboot и эмулируется в QEMU.

Код написан на C и NASM, активно развивается, имеет стабильные релизы и дружелюбную документацию. Проект открыт для вклада — каждый может попробовать себя в роли разработчика ОС!

Особенности

  • ядро Поддержка x86_64 (длинный режим) с базовым управлением памятью
  • загрузчик Совместимость с GRUB через Multiboot (загрузочный модуль написан на NASM)
  • драйверы Базовый видеовыход (текстовый режим VGA) и PS/2-клавиатура
  • отладка Встроенная поддержка отладки через GDB (символы DWARF)
  • инструменты Кросс-компилятор x86_64-elf-gcc + скрипты сборки для Linux, macOS, Windows (WSL, Cygwin, MSYS2) и Docker
  • стиль Модульная архитектура, чистый код с комментариями

Системные требования

Для сборки и запуска KwasOS вам понадобятся:

  • QEMU (рекомендуется версия 6.0+) — эмулятор для запуска ОС
  • GCC (кросс-компилятор для x86_64-elf) и binutils (с целевой платформой x86_64-elf)
  • NASM (ассемблер) версии 2.15+
  • Make (для использования Makefile)
  • Docker (опционально, для контейнеризованной сборки)

Поддерживаемые хост-системы: Linux (любой дистрибутив), macOS (с Homebrew), Windows (через WSL2, Cygwin или MSYS2).

Установка и подготовка инструментов

В репозитории есть подробные инструкции для каждой ОС. Приведу основные шаги:

Linux (Debian/Ubuntu)

sudo apt update sudo apt install build-essential nasm qemu-system-x86 gcc-multilib # Установка кросс-компилятора (если нужен отдельно) sudo apt install gcc-x86-64-linux-gnu binutils-x86-64-linux-gnu

macOS (Homebrew)

brew install qemu nasm gcc@13 binutils # Настройка PATH для кросс-компилятора (если установлен через brew) export PATH="/usr/local/opt/gcc@13/bin:$PATH"

Windows (WSL2)

Установите WSL2 с дистрибутивом Ubuntu и выполните команды для Linux.

Docker (универсальный способ)

# Сборка образа (если есть Dockerfile в репозитории) docker build -t kwasos-builder . # Запуск сборки внутри контейнера docker run --rm -v $(pwd):/build kwasos-builder make

После установки инструментов клонируйте репозиторий:

git clone https://github.com/frendalkir-alt/KwasOS.git cd KwasOS

🔨 Сборка проекта

Сборка осуществляется с помощью Makefile. Основные цели:

  • make all — полная сборка (ядро + загрузчик)
  • make clean — очистка объектных файлов
  • make run — сборка и запуск в QEMU
  • make debug — сборка с отладочной информацией и запуск с GDB-сервером

Пример сборки:

make all

Если у вас установлен кросс-компилятор в нестандартном месте, укажите переменные CC и AS:

make CC=x86_64-elf-gcc AS=nasm

Сборка создаёт образ kwasos.iso (или kernel.bin), готовый для загрузки.

Запуск в QEMU

После успешной сборки выполните:

make run

Эта команда запустит QEMU с эмуляцией x86_64, подключит образ и загрузит KwasOS. Вы увидите загрузочное сообщение и приглашение командной строки (если реализован shell).

Для отладки с GDB:

make debug

В другом терминале подключитесь к GDB-серверу:

gdb -ex "target remote localhost:1234"

Доступны все стандартные команды GDB (точки останова, пошаговое выполнение, просмотр регистров и памяти).

Архитектура системы

KwasOS построена по модульному принципу. Основные компоненты:

КомпонентОписаниеЯзык
BootloaderЗагрузчик (Multiboot-совместимый), инициализирует защищённый режим и передаёт управление ядруNASM
Ядро (Kernel)Управление памятью, обработка прерываний, планировщик (в разработке), системные вызовыC
ДрайверыВидеодрайвер (VGA текст), клавиатура (PS/2), таймер (PIT)C + inline asm
БиблиотекиСтандартные функции printf, malloc (simple), работа со строкамиC
Скрипты сборкиMakefile, линкер-скрипт (linker.ld) для размещения секцийMake / LD

Ядро загружается по физическому адресу 0x100000, использует страничную организацию памяти (4KB страницы). Прерывания обрабатываются через IDT (таблица дескрипторов прерываний).

Структура каталогов репозитория:

  • boot/ — загрузочный ассемблерный код и скрипты
  • kernel/ — исходники ядра (C)
  • drivers/ — драйверы устройств
  • lib/ — общие библиотечные функции
  • include/ — заголовочные файлы
  • build/ — временные объектные файлы (генерируется при сборке)

API ядра (базовые функции)

На данный момент реализован минимальный набор системных вызовов и утилит:

  • void print(const char *str) — вывод строки на экран
  • void print_hex(uint64_t value) — вывод числа в шестнадцатеричном виде
  • void *malloc(size_t size) — выделение памяти (простейший аллокатор)
  • void free(void *ptr) — освобождение памяти
  • void enable_interrupts() / disable_interrupts() — управление прерываниями
  • void putchar(char c) — вывод символа с обработкой управляющих кодов

Эти функции доступны из любого модуля ядра после включения соответствующих заголовков (#include <kernel.h>, #include <string.h> и т.д.). В будущем планируется добавить полноценные системные вызовы для пользовательских процессов.

Разработка и тестирование

Для удобства разработки в репозитории предусмотрены:

  • unit-тесты (в папке tests/) — проверка библиотечных функций (запуск на хосте, а не в эмуляторе)
  • скрипт проверки стиля (check-style.sh) — основан на clang-format
  • CI/CD (планируется) — автоматическая сборка при каждом коммите

Как добавить новую функцию:

  1. Создайте файл в соответствующем каталоге (например, kernel/new_feature.c).
  2. Объявите прототип в заголовочном файле (include/).
  3. Добавьте файл в Makefile (переменная OBJS).
  4. Соберите и протестируйте в QEMU.
  5. Напишите тесты (если применимо).

Перед отправкой pull request убедитесь, что код компилируется без предупреждений и проходит все тесты.
